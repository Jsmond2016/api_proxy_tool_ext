# 文档编写规范

当用户提出编写文档需求，且未明确要求格式或语言时，遵循以下规范。

## 核心规范

### 1. 默认语言

一律使用中文编写文档，包括但不限于：

- 文档标题
- 章节标题
- 正文内容
- 图表标题
- 注释说明

除非用户明确要求使用其他语言，或文档内容本身包含专有名词（API 名、库名、变量名等）。

### 2. 文档信息头

每份文档开头必须补充标准化的文档信息头，格式如下：

```markdown
# {文档标题}

> **编写时间**：{yyyy-MM-dd}；**使用模型**：{当前模型名称}；**用户**：{git user.name}

---
```

其中：

- **编写时间**：使用 `date +%Y-%m-%d` 格式，取当前日期
- **使用模型**：使用当前 AI 模型的名称（如 Claude Sonnet 4.6）
- **用户**：通过 `git config user.name` 获取，如果获取失败则显示"未知"

### 3. 触发场景

以下场景自动应用本规范：

- 需求文档 / 产品需求
- 技术方案 / 架构设计
- 接口文档 / API 文档
- 项目说明 / README
- 操作指南 / 用户手册
- 会议纪要 / 讨论总结
- 调研报告 / 技术选型
