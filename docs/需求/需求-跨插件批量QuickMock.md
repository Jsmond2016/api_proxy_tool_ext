# 跨插件批量QuickMock需求文档

> **编写时间**：2026-05-28；**使用模型**：GPT-5；**用户**：Jsmond2016

---

## 背景与目标

当前项目已经支持在扩展内部维护和管理 Mock 接口，但在实际使用中，异常接口的发现和筛选往往发生在另一个浏览器插件中。为了减少人工复制和逐条录入成本，需要支持由外部插件直接将一批接口 URL 发送给当前插件，由当前插件自动补全信息并生成对应的 Quick Mock 列表。

本需求的目标是打通“外部插件批量选择接口 -> 当前插件自动生成 Mock 列表 -> 打开新 Tab 展示结果”的完整流程，提升异常接口批量处理效率。

## 需求清单

1. 支持外部 Chrome 插件通过扩展消息调用当前插件
2. 外部插件发送的数据只要求包含勾选后的接口 URL 列表
3. 当前插件收到 URL 列表后，需要先做 URL 去重和标准化
4. 当前插件需要根据 URL 自动补全接口完整信息
5. 补全信息时优先使用当前插件本地已有接口配置
6. 若本地不存在，则尝试从当前配置的 Apifox 数据中补全
7. 若 Apifox 中也不存在，则仍允许用基础信息生成接口
8. 当前插件完成处理后，需要自动打开新的 Tab 展示本次结果
9. 新 Tab 需要默认切换到本次生成的 Mock 模块
10. 处理完成后，需要向外部插件返回统一的结果状态和提示文案

## 功能范围

### 包含

- 跨插件消息入口
- URL 去重与标准化
- 本地配置与 Apifox 信息补全
- 批量接口写入固定模块
- 打开新的 options Tab 展示结果
- 返回成功 / 失败 / 部分成功状态
- 联调说明文档

### 不包含

- 外部插件的 UI 开发
- 当前插件对外部插件的安装检测
- 历史批次保留与归档
- 多批次并行任务管理
- 细粒度权限控制页面

## 角色说明

### 外部插件 B

- 负责勾选异常接口
- 负责采集接口 URL 列表
- 负责调用当前插件的外部消息接口
- 负责根据返回结果提示用户

### 当前插件

- 负责接收外部请求
- 负责批量补全接口信息
- 负责生成并展示本次 Quick Mock 列表
- 负责返回统一结果

## 核心流程

1. 用户在外部插件 B 中勾选多个异常接口
2. B 插件提取接口 URL，并在发送前做一次去重
3. B 插件向当前插件发送 `BATCH_QUICK_MOCK` 消息
4. 当前插件接收消息后，对 URL 做二次去重和标准化
5. 当前插件依次根据 URL 查找本地配置和 Apifox 数据
6. 当前插件生成完整接口列表并写入固定批量模块
7. 当前插件自动打开新的 options Tab
8. 新 Tab 自动切换到本次批量模块并展示摘要
9. 当前插件向 B 插件返回处理结果
10. B 插件提示用户处理状态

## 业务规则

### URL 规则

- 只以接口路径作为匹配依据
- 忽略 query 和 hash
- 去除重复斜杠
- 统一补齐前导 `/`
- 同一路径只保留一条

### 信息补全规则

- 优先使用本地接口配置
- 本地无结果时，尝试使用 Apifox 信息补全
- Apifox 中无结果时，使用基础字段兜底创建

### Apifox 缺失策略

若接口在 Apifox 中不存在：

- 不视为失败
- 不展示 Apifox 链接、标签等附加信息
- 接口名称可回退为 URL
- 仍应正常加入本次 Quick Mock 列表

### 模块规则

- 本次批量结果写入固定模块 `quick.mock.external`
- 后一次批量结果覆盖前一次该模块数据
- 该模块仅用于承载最近一次外部批量导入结果

## 交互要求

### 外部插件调用结果提示

- 全部成功：提示 `操作成功`
- 全部失败：提示 `操作失败`
- 部分成功：提示 `部分操作成功，请前往 mock 列表查看`

### 当前插件新 Tab 展示

- 自动切换到批量模块
- 自动勾选本次成功导入的接口
- 自动高亮第一条成功接口
- 顶部展示任务摘要

摘要至少包含：

- 总数
- 成功数
- 失败数
- 未匹配到 Apifox 的接口数量
- 请求 ID

## 验收标准

- [ ] 外部插件可通过扩展消息成功调用当前插件
- [ ] 传入重复 URL 时，当前插件最终仅保留一条
- [ ] 本地已存在接口时，能正确复用本地信息
- [ ] 本地不存在但 Apifox 存在时，能自动补全接口信息
- [ ] Apifox 不存在时，仍可创建基础接口并计为成功
- [ ] 当前插件处理完成后，能自动打开新的 options Tab
- [ ] 新 Tab 能自动定位到 `quick.mock.external` 模块
- [ ] 返回状态和提示文案符合统一规则
- [ ] 联调文档可支持 B 插件完成对接

## 风险与后续优化

### 当前风险

- 当前批量模块采用覆盖模式，不保留历史批次
- 开发联调阶段默认允许所有扩展 ID 连接，正式发布前需要收紧白名单
- Apifox 数据补全依赖当前配置的 `apifoxUrl` 可访问

### 后续优化建议

- 增加批次归档能力
- 支持按时间或来源插件保留多份批量结果
- 增加固定白名单配置能力
- 增加更细粒度的失败原因展示
