# Chrome Web Store 发布准备文档

适用于 Manifest V3 扩展「API Proxy Tool」的 Chrome 网上应用商店发布指南。

---

## 一、前置条件

### 1.1 开发者账号

- 注册地址：https://chrome.google.com/webstore/devconsole
- 使用 Google 账号登录
- 需支付一次性注册费（约 $5 USD）
- 注册后即可管理扩展发布

### 1.2 环境要求

```bash
node >= 22.17.0
pnpm >= 10.5.2
```

---

## 二、构建与打包

### 2.1 生产构建

```bash
pnpm build:chrome
```

构建产物输出到 `dist_chrome/` 目录。

构建过程会自动：

- 编译 TypeScript → JavaScript
- 打包 React 组件和样式
- 生成 Manifest V3 配置
- 压缩图标资源
- 清除开发环境文件（.vite 等）

### 2.2 打包 ZIP

```bash
cd dist_chrome && zip -r ../extension-v$(node -p "require('../package.json').version").zip . && cd ..
```

注意事项：

- 必须在 `dist_chrome/` 目录内执行 zip 命令，避免 ZIP 包内包含外层目录前缀
- ZIP 包根目录必须直接包含 `manifest.json`

## 三、商店信息填写

### 3.1 基础信息

| 字段         | 要求                              | 内容                                                       |
| ------------ | --------------------------------- | ---------------------------------------------------------- |
| **扩展名称** | 与 `manifest.json` 中 `name` 一致 | API Proxy Tool                                             |
| **摘要**     | 132 字符以内，一句话描述          | API 请求拦截与 Mock 数据管理工具，支持声明式网络请求重定向 |
| **详细描述** | 详细功能列表，建议分项说明        | 参见下方模板                                               |
| **类别**     | 从商店分类中选择                  | Developer Tools                                            |
| **语言**     | 扩展界面语言                      | 简体中文 (zh-CN)                                           |

### 3.2 详细描述模板

**英文描述** — 参考 `docs/edge-market-description.md` 中的英文版。

**中文描述（推荐使用）**：

```
API Proxy Tool 是一款专为前端开发设计的浏览器扩展，通过声明式网络请求拦截和重定向功能简化开发流程。

核心功能：

主要特性
- 智能 API 拦截：自动拦截 API 请求并重定向到指定的 Mock 服务器
- 模块化管理：将 API 配置组织到多个模块，支持不同项目或功能分类
- 灵活匹配：支持包含匹配、精确匹配和正则表达式多种 URL 匹配方式
- 响应控制：配置响应延迟以模拟真实网络条件
- 直接 Mock 响应：无需外部服务器即可直接返回 Mock 数据

高级管理
- 批量操作：一键启用/禁用整个模块或单个 API
- 实时搜索：按名称或 URL 快速定位 API
- 配置同步：支持从 Apifox 同步 API 配置
- 全局控制：主开关控制所有代理功能

用户体验
- 全屏配置面板：在新标签页中打开综合配置面板
- 现代化界面：基于 Ant Design 构建的清晰直观界面
- 权限管理：生成和导出权限点用于后端集成

非常适合前端开发者使用。基于 Manifest V3 构建，遵循最小权限原则。

使用场景：
- 使用 Mock 数据进行前端开发
- API 测试和调试
- 微服务开发
- API 接口团队协作
- 快速原型制作和演示
```

### 3.3 权限用途说明

当前 `manifest.json` 中声明的权限及用途逐条说明：

| 权限                            | 用途说明                                                                                                        |
| ------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| `activeTab`                     | 获取当前活动标签页的 URL 信息，仅在用户点击扩展图标时临时生效，用于在配置页面中快速填充当前站点地址             |
| `storage`                       | 持久化存储用户的模块配置、API 规则和设置项；使用 `chrome.storage.sync` 在已登录同一 Google 账号的设备间同步配置 |
| `declarativeNetRequest`         | 使用声明式网络请求规则（非编程式）拦截和重定向 API 请求到 Mock 服务器，无需读取请求/响应体                      |
| `declarativeNetRequestFeedback` | 获取网络请求规则的匹配结果和调试信息，用于在界面中展示请求被哪些规则匹配                                        |

**host_permissions 说明**：

| 权限         | 用途说明                                                                           |
| ------------ | ---------------------------------------------------------------------------------- |
| `<all_urls>` | 用户可在配置中自定义需要拦截的域名范围，扩展仅在用户明确配置的域名和路径下拦截请求 |

### 3.4 数据隐私说明

```
数据收集与使用声明：

1. 本扩展不会收集、上传或分享任何用户数据到远程服务器。
2. 所有数据仅在用户本地浏览器中处理，不会离开用户的设备。
3. 用户配置通过 chrome.storage.sync 保存，仅用于在已登录同一 Google 账号的浏览器之间同步配置（需用户登录 Chrome 并启用同步功能）。
4. 本扩展不会读取或存储用户的浏览历史、书签、密码或其他个人信息。
5. 本扩展不会向任何第三方服务发送数据。
6. 本扩展不执行任何远程代码，所有代码均打包在扩展包内。
```

## 四、图片素材要求

### 4.1 扩展图标

| 尺寸    | 文件           | 说明                     |
| ------- | -------------- | ------------------------ |
| 128x128 | `icon-128.png` | 商店列表、详情页展示图标 |
| 32x32   | `icon-32.png`  | 浏览器工具栏、标签页图标 |

### 4.2 商店截图

| 规格 | 要求                   |
| ---- | ---------------------- |
| 尺寸 | 1280x800 PNG           |
| 数量 | 至少 1 张，建议 3-5 张 |
| 内容 | 展示扩展核心功能和界面 |

截图建议顺序：

1. **主界面** — 弹出面板 / 配置页完整界面
2. **模块管理** — 展示多模块配置和列表视图
3. **规则编辑** — 展示添加/编辑 API 规则的界面
4. **Apifox 同步** — 展示从 Apifox 同步配置的功能
5. **设置页面** — 展示全局设置和权限管理

提示：截图不能包含任何扩展的界面元素（如浏览器边框），仅截取扩展页面本身。

## 五、提交流程

### 5.1 操作步骤

1. 登录 Chrome Web Store Developer Dashboard
2. 点击 "New item" 上传 ZIP 包
3. 填写商店信息（名称、描述、类别、语言）
4. 上传图标和截图
5. 填写权限用途说明
6. 填写数据隐私说明
7. 选择发布范围（公开/非公开）
8. 提交审核

### 5.2 审核周期

- 首次提交：通常数小时到 1 个工作日
- 更新提交：通常数小时
- 自动化审核占多数，复杂情况可能进入人工审核

### 5.3 常见拒绝原因及处理

| 拒绝原因           | 解决方案                                 |
| ------------------ | ---------------------------------------- |
| 权限请求过多       | 移除不必要的权限，确保每个权限有明确用途 |
| 权限用途说明不充分 | 逐条补充具体代码层面的使用场景           |
| 功能描述与实际不符 | 确保描述准确反映扩展当前功能             |
| 截图不符合要求     | 调整尺寸至 1280x800，去除无关界面元素    |

## 六、版本更新流程

```bash
# 1. 升级版本号
pnpm version:patch   # 补丁版本：1.5.29 → 1.5.30
pnpm version:minor   # 次要版本：1.5.29 → 1.6.0
pnpm version:major   # 主要版本：1.5.29 → 2.0.0

# 2. 构建
pnpm build:chrome

# 3. 打包
cd dist_chrome && zip -r ../extension-v$(node -p "require('../package.json').version").zip . && cd ..

# 4. 登录商店控制台上传新 ZIP 包
```
